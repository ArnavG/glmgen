Fit a trendfilter model. Fits a trendfilter model for a sequence of lambda tuning parameters. The user must supply allocated memory to store the output, with the function itself returning only void.

Parameters
ya vector of responses
xa vector of response locations; must be ordered
wa vector of sample weights
nthe length of y, x, and w
kdegree of the trendfilter; i.e., k=1 linear
familyfamily code for the type of fit; family=0 for OLS
max_itermaximum number of ADMM interations; ignored for k=0
lam_flag0/1 flag for whether lambda sequence needs to be estimated
lambdaeither a sequence of lambda when lam_flag=0, or empty allocated space if lam_flag=1
nlambdanumber of lambda values; need for both lam_flag=0 and 1
lambda_min_ratiominimum ratio between min and max lambda; ignored for lam_flag=0
betaallocated space of size n*nlambda to store the output coefficents
objallocated space of size max_iter*nlambda to store the objective
iterallocated space of size nlambda to store the number of iterations
statusallocated space of size nlambda to store the status of each run
rhotuning parameter for the ADMM algorithm; set to 1 for default
obj_tolstopping criteria tolerance; set to 1e-10 for default
alpha_lsfor family != 0, line search tuning parameter; set to 0.5 for default
gamma_lsfor family != 0, line search tuning parameter; set to 0.8 for default
max_iter_lsfor family != 0, max number of iterations in line search; set to 50 for default
max_inner_iterfor family != 0, max number of iterations in inner ADMM; set to 250 for default
verbose0/1 flag for printing progress
Returns
void

Default call to tf_admm Example of how to call tf_admm, taking only the response vector and observation size
as inputs. Users will likely need to adjust tf_admm_default for their own needs, but it serves as a starting template.

Parameters
ya vector of responses
nthe length of y, i.e., the number of observations
Returns
Returns a pointer to beta values, a column oriented (nlambda x n) array.
Note
The user is responsible for freeing the array pointed to by the response.

Low level fitting routine for a Gaussian trendfiltering problem. Function used by tf_admm to fit a Gaussian ADMM trendfilter, or as a subproblem by tf_admm_glm when using logistic or poisson losses. Fits the solution for a single value of lambda.

Parameters
ya vector of responses
xa vector of response locations; must be ordered
wa vector of sample weights
nthe length of y, x, and w
kdegree of the trendfilter; i.e., k=1 linear
max_itermaximum number of ADMM interations; ignored for k=0
lamthe value of lambda
betaallocated space for output coefficents; must pre-fill as it is used in warm start
alphaallocated space for ADMM alpha covariates; must pre-fill as it is used in warm start
uallocated space for ADMM u covariates; must pre-fill as it is used in warm start
objallocated space to store the objective; will fill at most max_iter elements
iterallocated space to store the number of iterations; will fill just one element
statusallocated space of size nlambda to store the status of each run
rhotuning parameter for the ADMM algorithm; set to 1 for default
obj_tolstopping criteria tolerance; set to 1e-10 for default
DktDkpoint to the inner product of DktDk
verbose0/1 flag for printing progress
Returns
void
